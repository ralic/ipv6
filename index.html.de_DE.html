Wie dieser Test funktioniert: Ihr Webbrowser wird angewiesen eine Reihe von URLs aufzurufen. Aus der Auswertung der erfolgreichen und fehlgeschlagenen Aufrufen lässt sich ableiten wie bereit Sie sind, wenn Inhaltsanbieter beginnen Ihre Websiten via IPv6 verfügbar zu machen.

Hier klicken für Durchgeführte Tests

Test mit IPv4 DNS-Eintrag
ausstehend
-
Ruft ein Objekt auf, welches nur über einen A-Eintrag im DNS verfügt. Es wird eine IPv4-Verbindung erwartet. Nur-IPv6-Benutzer können unter Umständen auch eine Verbindung herstellen, nämlich dann, wenn ihr ISP eine NAT64/DNS64- oder eine andere Proxylösung zur Verfügung stellt.
Test mit IPv6 DNS-Eintrag
ausstehend
-
Ruft ein Objekt auf, welches nur über einen AAAA-Eintrag im DNS verfügt. Es wird eine IPv6-Verbindung erwartet. Für Benutzer ohne IPv6-fähigen Internetzugang wird dieser Test fehlschlagen. Solange dies ohne grosse Verzögerung geschieht, ist dies zurzeit noch in Ordnung.
Test mit Dual Stack DNS-Eintrag
ausstehend
-
Dies ist der wichtigste Test. Es wird überprüft, ob Ihr Browser Webseiten aufrufen kann, welche sowohl via IPv4, als auch IPv6 erreichbar sind. Systeme, welche nur über IPv4-Konnektivität verfügen, sollten solche Seiten problemlos erreichen können (via IPv4).

Falls dieser Test fehlschlägt oder in einer Zeitüberschreitung ended, werden Sie schwerwiegende Probleme bekommen, wenn Inhaltsanbieter beginnen, ihre Webseiten via IPv6 verfügbar zu machen.
Test mit Dual Stack und große Pakete
ausstehend
-
Es wird überprüft ob Ihr Browser einen Dual Stack Server erreichen kann (wie beim Test mit Dual Stack DNS Eintrag) und ob große Pakete über die Verbindung gesendet und empfangen werden können. Falls dieser Test aus irgendeinem Grund mit Time-Out fehlschlägt, ist mit Problemen am World IPv6 Tag zu rechnen.
Test IPv4 ohne DNS
ausstehend
-
Dieser Test versucht eine Verbindung direkt zu einer IPv4-Adresse zu erstellen (ohne DNS). Dies sollte für die meisten Systeme keine Probleme darstellen, es sei denn, Sie verfügen nur über einen IPv6-Internetzugang. Falls der erste Test funkioniert, dieser aber fehlschlägt, bestätigt dies, dass der Provider NAT64/DNS64 einsetzt. In diesem Fall müssen Sie Webseiten über ihren Namen aufrufen und nicht direkt über ihre IP-Adresse.
Test IPv6 ohne DNS
ausstehend
-
Dieser Test versucht eine Verbindung direkt zu einer IPv6 Adresse zu erstellen (ohne DNS). Der Hauptgrund für diesen Test ist es Ihre IPv6 Konnektivität unabhängig von der IPv6 Namensauflösung zu testen. Ein weiterer Zweck ist zu prüfen ob Teredo aktiviert ist. Einige Systeme benutzen Teredo nur falls eine IPv6 Adresse im URL angeben wird.
Test große IPv6-Pakete
ausstehend
-
Stellt sicher, dass IPv6-Anfragen mit grossen Paketen funktionieren. Falls dieser Test in einer Zeitüberschreitung ended, andere IPv6-Tests jedoch funktionieren, liegt vermutlich ein Problem mit der PMTUD (Path MTU Detection) vor. Tritt oft im Zusammenhang mit IP-Tunneln auf. Überprüfen Sie, dass ICMPv6 Typ 2 ("Paket zu groß") Nachrichten nicht durch Ihre Firewall gefiltert werden.
Test ob der DNS-Server des ISP IPv6 unterstützt
ausstehend
-
(Dies gibt Zusatzpunkte)
Dies ist eine Überprüfung des DNS Servers Ihres ISPs. Falls dieser Test erfolgreich ist, bedeutet dies dass der DNS Server fähig ist, nur via IPv6 erreichbare, authorative DNS Server im Internet zu erreichen. Dies ist (zumindest im Moment) nicht unbedingt erforderlich um Webseiten über eine IPv6 Verbindung zu erreichen.
IPv4-Dienstleister finden
ausstehend
-
Versucht, zu erkennen, welcher Internet-Service-Provider für IPv4 verwendet wird. Dies kann von dem Markennamen der Firma abweichen, mit der Sie einen Vertrag haben oder eine alte Firmenbezeichnung sein. Der gezeigte Name spiegelt wieder, wie Ihr ISP in der Netzbetreiber-Community bekannt ist.
IPv6-Dienstleister finden
ausstehend
-
Versucht, zu erkennen, welcher Internet-Service-Provider für IPv6 verwendet wird. Wenn dieser Name nicht mit dem ISP für IPv4 übereinstimmt, verwenden Sie evtl. einen IPv6 Tunnel oder einen Drittanbieter.
Test um fehlerhafte DNS zu erkennen
ausstehend
-
Teste auf fehlerhafte AAAA-Record. Einige Router interpretieren als "A"-Records und geben nur die ersten 32 Bit zurück. Dieser Test darf keine erfolgreiche Verbindung erzeugen.

Die Javascript basierten Überprüfungen scheinen fehlgeschlagen zu sein. Dies könnte von einer Browser Erweiterung verursacht worden sein. Bitte geben Sie an welche Erweiterungen Sie in diesem Webbrowser aktiviert haben.
Falls Sie über genügend technisches Verständnis verfügen, fügen Sie bitte die Ausgabe folgender Kommandos ein:
netstat -nr
ifconfig -a
Kopieren Sie die Ausgabe dieser Kommandos in das untenstehende Feld ein. Dies kann uns helfen bei der Interpretation Ihrer Testergebnisse.
Falls Sie über genügend technisches Verständnis verfügen, fügen Sie bitte die Ausgabe folgender Kommandos ein:
ip -f inet6 route show
ip -f inet6 addr
Kopieren Sie die Ausgabe dieser Kommandos in das untenstehende Feld ein. Dies kann uns helfen bei der Interpretation Ihrer Testergebnisse.
Falls Sie über genügend technisches Verständnis verfügen, öffnen Sie die "Eingabeaufforderung" oder "cmd" und führen diese drei Kommandos aus:
ipconfig /all
netsh interface ipv6 show teredo
netsh interface ipv6 show route
Kopieren Sie die Ausgabe dieser Kommandos in das untenstehende Feld ein. Dies kann uns helfen bei der Interpretation Ihrer Testergebnisse.
