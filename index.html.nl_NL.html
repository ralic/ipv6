Hoe deze test werkt: Uw browser probeert om een aantal URL's te benaderen. De combinatie van gelukte en mislukte pogingen vertelt hoe gereed u bent voor het moment dat websites via IPv6 beschikbaar zijn.

Bekijk Uitgevoerde tests

Test met een IPv4 DNS record
in afwachting
-
Benader een object met slechts een A record in DNS. Hierbij wordt naar verwachting gebruik gemaakt van IPv4. Gebruikers met alleen IPv6 kunnen dit record mogelijk bereiken als hun ISP een NAT64/DNS64 of proxy aanbiedt.
Test met een IPv6 DNS record
in afwachting
-
Benader een object met slechts een AAAA record in DNS. Hierbij wordt naar verwachting gebruik gemaakt van IPv6. Gebruikers zonder IPv6 zullen deze test naar verwachting zien mislukken. Zo lang als deze test binnen korte tijd mislukt is alles -voor nu- in orde.
Test met een Dual Stack DNS record
in afwachting
-
Dit is de belangrijkste test. Deze test controleert of uw browser kan verbinden met een website die zowel IPv4 als IPv6 DNS records aanbiedt. Machines met alleen IPv4 zouden zonder problemen (via IPv4) moeten kunnen verbinden.

Als deze test mislukt of een timeout geeft kunt u grote problemen verwachten als websites via IPv6 beschikbaar gemaakt worden.
Test met een Dual Stack DNS en grote pakketten
in afwachting
-
Deze test controleert of u een dual-stack server kan benaderen en dat u grote pakketten kan verzenden en ontvangen over die verbinding. Indien deze test mislukt, geeft dat problemen aan voor World IPv6 Day.
Test IPv4 zonder DNS
in afwachting
-
Deze test zal proberen een IPv4-adres direct te benaderen. Deze succes zal slagen voor iedereen, tenzij ze alleen IPv6 gebruiken. Als de eerste test slaagt maar deze mislukt is het waarschijnlijk dat uw provider NAT64/DNS64 gebruikt en moet u proberen alleen te verbinden op hostnamen en niet op IP-adressen.
Test IPv6 zonder DNS
in afwachting
-
Deze test zal proberen een IPv6-adres direct te benaderen. Het belangrijkste doel van deze test is om te bepalen of uw mogelijkheid om via IPv6 te verbinden onafhankelijk is van de mogelijkheid om DNS ervoor op te vragen. Een tweede doel is om te zien of Teredo gebruikt wordt, sommige systemen gebruiken Teredo alleen als er een IPv6-adres in het URL staat.
Test IPv6 met grote pakketten
in afwachting
-
Deze test controleert of IPv6 requests met grote pakketten werken. Als deze test een timeout geeft maar andere testen werken duidt dit erop dat er PMTUD problemen zijn, mogelijk veroorzaakt door IP tunnels. Double check to make sure that ICMPv6 Type 2 ("Packet Too Big") messages are not filtered by your firewall.
Test of uw ISP's DNS server IPv6 gebruikt
in afwachting
-
(Dit is bonus credit)
Dit is een test van de DNS resolver van uw ISP (in plaats van een test van uw machine). Indien deze test lukt, is uw DNS server (vaak beheerd door uw ISP) in staat om een authoritatieve DNS-server met alleen IPv6 te benaderen. Dit is -voor nu- niet kritiek om websites via IPv6 te benaderen.
Vind IPv4 Service Provider
in afwachting
-
Poging om je Internet Service Provider te identificeren voor IPv4. De naam kan verschillen van de commerciële naam die jij kent als jouw internet aanbieder; of het kan een vorige naam zijn van het bedrijf. Het bedrijf is bekend in de netwerk operatorengemeenschap onder de naam die wordt weergegeven.
Vind IPv6 Service Provider
in afwachting
-
Poging om je Internet Service Provider te identificeren voor IPv6. Wanneer de naam bij IPv4 en de naam bij IPv6 verschilt, dan kan het zijn dat je gebruikt maakt van een zogenaamde "tunnel"; of een andere manier zoals een derde aanbieder voor IPv6.
Test voor kapotte DNS
in afwachting
-
Deze test doet verzoeken voor een fout AAAA record. Sommige routers behandelen deze als "A" records en bewaren alleen de eerste 32 bits. We verwachten dat deze test geen verbinding kan maken.
