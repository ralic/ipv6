這個測試是如何運作的 ︰ 您的瀏覽器將會連接一系列的網址。各項測試結果統計將顯示當網站啟用 IPv6 時你準備好了多少。

點擊查看 測試結果

測試 IPv4 DNS 記錄
等待中...
-
查詢一個只有 A 記錄在 DNS 主機上的網站。這是測試 IPv4 連線。只有 IPv6 的使用者如果網路服務提供者 (ISP) 使用 NAT64/DNS64 或代理伺服器的解決方案仍可能完成這項。
測試 IPv6 DNS 記錄
等待中...
-
查詢一個只有 AAAA 記錄在 DNS 主機上的網站。這是測試 IPv6 連線。用戶如果尚未設定 IPv6 網路連線將會看到這項測試失敗。現階段來說測試失敗不會受到影響。
測試雙協定 DNS 記錄
等待中...
-
這是最重要的測試。這將驗證您的瀏覽器可以連接到一個已經啟用 IPv4 和 IPv6 記錄的網站。只有 IPv4 的用戶也可以連線成功 （透過 IPv4 連線）。

如果這項測試失敗或超時，你可能在網站啟用 IPv6 時遇到一些問題。
測試雙協定 DNS 和較大封包
等待中...
-
驗證您可以連接到一個雙協定伺服器(如 ds 測試)；你可以透過該連線傳送/接收大數據封包。如果這個測試不知原因而逾時，它代表「世界 IPv6 日」的麻煩。
測試 IPv4 (跳過DNS)
等待中...
-
這項測試將試著連線到一個 IPv4 位址。大部分用戶都可以通過這項測試，僅啟用 IPv6 連線的用戶除外。如果第一項測試通過，但這項測試失敗時，則可以確認你的網路提供者使用 NAT64/DNS64 技術；你只需要使用網址來連接網站而不是使用 IP 位址。
測試 IPv6 (跳過DNS)
等待中...
-
這項測試將試著連線到一個16進位的 IPv6 位址。這項測試主要是用來分別測試你的 IPv6 的直接能力，而不查詢 DNS 進行連線。次要為確認你是否有啟用 Teredo ；部份系統只會在 url 裡有 IPv6 地址時使用 Teredo 。
測試 IPv6 大包
等待中...
-
驗證用較大封包來處理 IPv6 要求是否正常運作。如果這項測試超時，但其他 IPv6 測試正常，這可能是PMTUD 問題，可能是由 IP 通道造成的。 請檢查並確認 ICMPv6 Type 2 (“封包太大”) 的訊息不是由防火牆阻擋的。
測試您的 ISP DNS 伺服器是否使用 IPv6
等待中...
-
（這是加分項目）
這項測試是針對你的 ISP (而不是測試你的主機)。如果這項測試通過，代表你的 DNS 伺服器(通常由你的 ISP 維護) 可以連結在網路上的 IPv6 DNS 認證伺服器。現階段這不會造成你透過 IPv6 連接網站。
查詢 IPv4 服務提供者
等待中...
-
這項測試是在辨識你的 IPv4 是由哪個 ISP 提供。這可能會與你所知的當地名稱有所不同；或可能會顯示舊的公司名稱，這個結果是直接顯示在網路營運者社群的資料。
查詢 IPv6 服務提供者
等待中...
-
這項測試是在辨識你的 IPv6 是由哪個 ISP 提供。如果你的 IPv4 與 IPv6 的 ISP 名稱不同，則可以得知你使用了通道技術；或是其他 IPv6 技術的第三方提供者。
對於越野 DNS 測試
等待中...
-
格式不正確的 AAAA 記錄進行查詢。某些路由器責難這些作為"A"記錄，並且只保留第 32 位。我們希望這個測試連接失敗。

如果總結結果指出的問題，你（或你的技術人員） 可以使用上面的資訊來診斷問題。每個測試 url 和它們的測試結果顯示在左邊。而在右邊，你可以看到該 URL 的測試說明。
