Come funziona questo test: Il tuo browser cercherà di raggiungere una serie di indirizzi URL. Questo permette di capire se la tua configurazione è in grado di supportare siti web su IPv6.

Clicca per visualizzare Verifiche Effettuate

Test con record DNS IPv4
in sospeso
-
Recupero di un oggetto che ha solo un record A nel DNS. Questo dovrebbe utilizzare l' IPv4. Gli utenti IPv6-only potrebbero ancora raggiungerlo, se il prorpio provider ha impiegato una soluzione NAT64/DNS6 o una soluzione proxy.
Test con record DNS IPv6
in sospeso
-
Recupero di un oggetto che ha solo un record AAAA nel DNS. Questo dovrebbe utilizzare l' IPv6. Gli utenti che non usano ancora l' IPv6 dovrebbero vedere questo fail. Finchè fallisce rapidamente, sarà OK - per adesso.
Test con record DNS Dual Stack
in sospeso
-
Questo è il test più importante. Il test verifica che il tuo browser possa connettersi a un sito che ha entrambi i record IPv4 e IPv6 pubblicati. Gli hosts che usano solo l'IPv4 dovrebbero connettersi senza problemi (usando l'IPv4).

Se questo test fallisce o va in timeout, puoi aspettarti problemi più grandi visto che i publishers hanno cominciato a offrire i loro siti tramite IPv6.
Test con record DNS Dual Stack e pacchetti grandi
in sospeso
-
Controlla che sia possibile connettersi a un server dual-stack (come nel test ds) e che sia possibile inviare/ricevere pacchetti di grandi dimensioni su questa connessione.
Test IPv4 senza DNS
in sospeso
-
Questo cercherà di connettersi usano un indirizzo numerico IPv4 letterale. Dovrebbe funzionare per la maggior parte delle persone, a meno che non stiano usando IPv6-only. Se il primo test ha funzionato, ma questo fallisce, potrebbe essere una conferma che il tuo provider sta usando NAT64/DNS64; sarà necessario connettersi usando solo nomi host invece degli indirizzi IP numerici.
Test IPv6 senza DNS
in sospeso
-
Questo cercherà di connettersi con un indirizzo esadecimale IPv6 letterale. Lo scopo principale di questo test è di separare la connettività IPv6 dall'abilità di recuperare DNS per essa. Il secondo scopo è di vedere se è attivo Teredo; alcuni sistemi possono usare solo Teredo quando un indirizzo IPv6 è nell'URL.
Test IPv6 con pacchetti grandi
in sospeso
-
Valida che le richieste IPv6 con pacchetti di grande dimensioni funzioni. Se il test va in time out, ma altri test IPv6 funzionano, suggerisce che potrebbero esserci dei problemi PMTUD; che forse coinvolgono dei tunnels IP. Controlla che i pacchetti ICMPv6 Type 2 ("Packet Too Big") non vengono filtrati dal tuo firewall.
Test per verificare se il server DNS del tuo provider utilizza IPv6
in sospeso
-
(Questo è credito bonus)
Questo è un test per il resolver del tuo ISP (invece di testare l' host). Se il test viene superato, il server DNS (spesso eseguito dal tuo ISP) è in grado di raggiungere i servers DNS authoritative per connettività IPV6-only su internet. Questo non è fondamentale (per il momento) per raggiungere i siti via IPv6.
Ricerca del provider di servizi IPv4
in sospeso
-
Tenta di identificare quale Internet Service Provider si utilizza per l'IPv4. Potrebbe essere diverso dal nome marketing scelto per il mercato; o potrebbe visualizzare il nome della precedente compagnia. Il nome mostrato rispecchia come viene chiamato nella comunità degli operatori di rete.
Ricerca del provider di servizi IPv6
in sospeso
-
Tenta di identificare quale Internet Service Provider si utilizza per l'IPv6. Quando il nome IPv4 e quello IPv6 non corrispondo, potrebbe indicare che si sta usando un tunnel; o qualche altra forma di provider di terze parti per l'IPv6.
Test per DNS buggy
in sospeso
-
Queries per un record AAAA non valido. Alcuni routers manomettono questi record con records di tipo A e mantengono solo i primi 32 bits. Si aspetta che il test fallisca per connettersi.

Se il riepilogo dei risultati visualizza dei problemi, tu (o il tuo supporto tecnico) dovresti essere in grado di usare le informazioni sopra per rilevare i problemi.
