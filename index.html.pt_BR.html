Testar sua conectividade IPv6. (Testar novamente)

JavaScript necessário

Este site requer JavaScript, bem como a capacidade de utilizar scripts entre sites, a fim de realizar os testes.

Se esta mensagem não desaparecer, significa que o JavaScript foi desativado por um plugin, extensão ou configuração explícita de seu navegador.

Você usa o NoScript?

Se você usa este complemento do Firefox, você vai precisar marcar a opção "Temporariamente permitir tudo nesta página". Você vai precisar fazer isso duas vezes para que tudo funcione. Ao invés disto, você pode desabilitar o NoScript completamente até que você termine de utilizar este site.

Você pode optar por visualizar o teste simples, que vai te mostrar rapidamente um teste de sucesso/falha para IPv4, IPv4+IPv6 e IPv6. No entanto, este teste oferece pouca informação para diagnóstico de conexão.

Como esse teste funciona: Seu navegador será instruído a acessar uma série de URLs. A combinação de sucessos e falhas dirá o quão preparado você está para o momento em que as entidades começarem a publicar seus sites em IPv6.

Clique para ver Testes Executados

Testa com um registro DNS IPv4
pendente
-
Busca um objeto que tem apenas um registro A no DNS. É esperado o uso de IPv4. Usuários que utilizam apenas IPv6 ainda podem acessar isso se seus provedores empregarem alguma solução envolvendo NAT64/DNS64 ou um proxy.
Testa com um registro DNS IPv6
pendente
-
Busca um objeto que tem apenas um registro AAAA no DNS. É esperado o uso de IPv6. Usuários que ainda não usam a Internet IPv6 provavelmente verão isso falhar. Enquanto falhar rapidamente, está tudo bem - para o momento.
Testa com um registro DNS duplo
pendente
-
Esse é o teste mais importante. Ele verifica se seu navegador consegue acessar sites que possuem tanto um registro IPv4 quanto um IPv6. Computadores baseados exclusivamente em IPv4 devem fazer isso sem problemas (usando IPv4).

Se esse teste falhar ou não for completado em tempo hábil, espere problemas quando entidades começarem a disponibilizar seus sites em IPv6.
Testa DNS duplo e pacote grande
pendente
-
Verifica o funcionamento de conexões a um servidor de pilha dupla (como o teste ds) e se você pode enviar/receber pacotes grandes nessa conexão. Se esse teste não for completado em tempo hábil, isso indica que você terá problemas no dia mundial do IPv6.
Testa IPv4 sem DNS
pendente
-
Isso tentará uma conexão usando um endereço IPv4, o que deve funcionar para a maioria das pessoas, a menos que estejam usando exclusivamente IPv6. Se o primeiro teste funcionou mas este não, isso pode significar que seu provedor está usando NAT64/DNS64. Você apenas precisará tentar conectar usando nomes ao invés de endereços IP.
Testa IPv6 sem DNS
pendente
-
Isso tentará fazer uma conexão usando um endereço IPv6 hexadecimal. A ideia é fazer a separação de sua conectividade IPv6 de sua capacidade de utilizar DNS para isso e depois verificar se você está utilizando Teredo. Alguns sistemas só podem usar Teredo quando há um endereço IPv6 na URL.
Testa pacote IPv6 grande
pendente
-
Verifica o funcionamento de requisições IPv6 usando pacotes grandes. Se esse teste não for concluído em tempo hábil mas outros testes IPv6 funcionarem, é provável que haja problemas com PMTUD, possivelmente envolvendo túneis IP. Verifique novamente se as mensagens ICMPv6 Tipo 2 (Pacote grande demais) não estão sendo filtradas pelo seu firewall.
Testa se o servidor DNS de seu provedor usa IPv6
pendente
-
(bônus)
Esse é um teste para o seu provedor e não para seu computador. Se esse teste funcionar, seu servidor DNS (frequentemente mantido em seu provedor) é capaz de acessar servidores DNS autoritativos baseados exclusivamente em IPv6. No momento não é essencial para você acessar sites via IPv6.
Encontrar provedor de serviços de IPv4
pendente
-
Tenta identificar qual Provedor de Serviços de Internet (ISP) você utiliza com o IPv4. Este nome pode ser diferente do nome comercial que você está acostumado ou pode representar um nome que esta empresa já utilizou no passado. O nome mostrado é como seu provedor é conhecido na comunidade de operadores de rede.
Encontrar provedor de serviços de IPv6
pendente
-
Tenta identificar qual Provedor de Serviços de Internet (ISP) você utiliza com o IPv6. Se o nome IPv4 e IPv6 são diferentes, isto pode sugerir que você está utilizando um túnel ou alguma forma de provedor terceirizado para IPv6.
Testa DNS problemático
pendente
-
Consultas por um registro AAAA malformado. Alguns roteadores os tratam inadequadamente como registros A e mantém apenas os primeiros 32 bits. Queremos que esse teste falhe.

Se o resumo dos resultados indicou problemas, você (ou seu suporte técnico) deve ser capaz de utilizar as informações acima para fazer os diagnósticos. Cada URL de teste e seus resultados aparecem no lado esquerdo. À direita você verá uma descrição do que aquela URL testa.

Depois que cada teste é executado. Se o resumo não faz sentido considerando os sintomas registrados acima ou se você precisa de assistência, por favor, sinta-se à vontade para nos contactar.
