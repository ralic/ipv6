Probar tu conectividad IPv6. (Prueba de nuevo)

JavaScript Requerido

Este sitio requiere JavaScript, así como la habilidad para tirar secuencias de comandos entre sitios, con el fin de realizar la prueba.

Si este mensaje no desaparece, significa que se ha desactivado JavaScript, por un plugin, una extensión en su navegador o por configuración explícita del navegador.

¿Utilizas NoScript?

Si utiliza este Add-on de Firefox, necesitará "Permitir temporalmente toda esta página". Usted necesitará hacer esto dos veces para que todo funcione. Alternativamente, desactive NoScript totalmente hasta que haya terminado con este sitio.

Usted puede optar en su lugar para ver la prueba sencilla, que le dará un rápido pase/fallo para IPv4, IPv6 y IPv4 + IPv6. Sin embargo ofrecerá poca información de diagnóstico.

Cómo funciona esta prueba: Su navegador recibirá instrucciones para llegar a una serie de URLs. La combinación de éxitos y fracasos cuenta una historia sobre lo listo que está para cuando editores comiencen a ofrecer sus sitios web sobre IPv6.

Click para ver Pruebas ejecutadas

Prueba con registro DNS IPv4
pendiente
-
Obtiene un objeto que tiene sólo un registro A en el DNS. Se espera que utilice IPv4. Los usuarios de sólo-IPv6 todavía podrían alcanzar esto, si su proveedor de servicios ha empleado una solución NAT64/DNS64 o proxy.
Prueba con registro DNS IPv6
pendiente
-
Obtiene un objeto que tiene un registro sólo AAAA en el DNS. Se espera que utilice IPv6. Los usuarios sin conexión a la red Internet IPv6 probablemente vean esta prueba fallar. Mientras falle rápidamente, todo irá bien - por ahora.
Prueba con registro de doble pila DNS
pendiente
-
Esta es la prueba más importante. Esto verifica que su navegador puede conectarse a un sitio que tiene registros de IPv4 e IPv6 publicados. Computadores con sólo IPv4 deben conectarse bien (usando IPv4).

Si esta prueba falla o el tiempo expira, usted puede esperar grandes problemas cuando editores comiencen a ofrecer sus sitios en IPv6.
Prueba de doble pila DNS y paquete grande
pendiente
-
Valida que se pueda conectar a un servidor dual-stack (como la prueba de ds); y que usted pueda enviar/recibir paquetes grandes en esa conexión. Si esta prueba expira por cualquier motivo, indica problemas para el día mundial de IPv6.
Prueba IPv4 sin DNS
pendiente
-
Esto intentará conectar con una dirección numérica literal de IPv4. Esto debería funcionar para la mayoría, a menos que use solo IPv6. Si la primera prueba pasa, pero esto no funciona, es probable que su proveedor está utilizando NAT64/DNS64; Usted necesitará conectarse usando nombres de host en lugar de direcciones IP numéricas.
Prueba IPv6 sin DNS
pendiente
-
Esto intentará conectarse con una dirección literal de IPv6 hexadecimal. El propósito principal de esta prueba es separar su conectividad en IPv6 de su habilidad para buscar DNS para él. Un objetivo secundario es ver si tiene Teredo activado; algunos sistemas sólo pueden usar Teredo cuando una dirección IPv6 se encuentra en la URL.
Prueba paquete grande de IPv6
pendiente
-
Valida que solicitudes IPv6 con grandes paquetes trabajen. Si esta prueba expira, pero otras pruebas IPv6 pasan, sugiere que puede haber problemas de PMTUD; posiblemente involucrando túneles IP. Compruebe para asegurarse que los mensajes de ICMPv6 tipo 2 ("Paquete demasiado grande") no sean filtrados por su firewall.
Prueba si el servidor DNS de su ISP utiliza IPv6
pendiente
-
(Esto es crédito extra)
Esta es una prueba del resolutor de su ISP (en lugar de una prueba de su computador). Si pasa esta prueba, el servidor DNS (a menudo operado por su ISP) es capaz de alcanzar servidores autoritativos DNS sólo IPV6 en Internet. Esto no es crítico (en este momento) para que usted pueda llegar a sitios vía IPv6.
Encontrar proveedor de servicios IPv4
pendiente
-
Intenta identificar qué proveedor de servicios de Internet utiliza para IPv4. Esto puede ser diferente al nombre de comercialización que ve en tu mercado local; o puede reflejar un nombre anterior de la empresa. El nombre mostrado refleja cómo es conocido en la comunidad de operador de red.
Encontrar proveedor de servicios IPv6
pendiente
-
Intenta identificar qué proveedor de servicios de Internet utiliza para IPv6. Cuando el nombre de IPv4 y el IPv6 no coinciden, puede sugerir que usted está usando un túnel; o alguna forma de tercero como proveedor de IPv6.
Prueba de DNS pulgoso
pendiente
-
Consultas para un registro con formato incorrecto de AAAA. Algunos routers manipulan estos como registros "A" y sólo mantienen los primeros 32 bits. Queremos que esta prueba falle en la conexión.

Si el resumen de resultados de los problemas indicados, usted (o su soporte técnico) puede utilizar la información de arriba para diagnosticar los problemas. Cada uno de los urls de prueba y sus resultados se muestran en el lado izquierdo. A la derecha verá una descripción de lo que ese URL fue diseñado para probar.

Después de que cada prueba haya corrido. La página de resumen intenta ver los resultados. Si el resumen no parece tener sentido dado los síntomas registrados anteriormente, o si necesitas más ayuda, por favor no dude en contactarnos.
